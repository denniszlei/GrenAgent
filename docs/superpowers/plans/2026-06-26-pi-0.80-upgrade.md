# Pi SDK 升级实现计划（0.79.x → 0.80.2）

> **面向 AI 代理的工作者：** 推荐用 superpowers:executing-plans 逐任务实现此计划。步骤使用复选框（`- [ ]`）语法跟踪进度。本计划无需 TDD 新增业务测试，重在"改依赖 + 迁移 import + 重建 sidecar + 跑既有验证"。

**目标：** 把底层 `@earendil-works/pi-*` SDK（`pi-ai` / `pi-coding-agent` / 传递依赖 `pi-agent-core` / `pi-tui`）从 0.79.x 升到 npm 最新 `0.80.2`，并完成 0.80.0 带来的 `pi-ai` 根入口 API 迁移，保证 sidecar 二进制 typecheck、测试、运行时均正常。

**背景：** "Pi"即本项目运行时所依赖的 pi 编码代理 SDK。当前 `extensions/` 解析 0.79.10、`cli/` 锁 0.79.6；npm `latest = 0.80.2`（0.80.0 于 2026-06-23 发布）。

**技术栈：** TypeScript；包管理器三套并存——`extensions/`（bun）、`cli/`（npm，pi-coding-agent 带 `hasShrinkwrap`）、`tauri-agent/`（pnpm，仅消费编译出的 sidecar 二进制）。构建链见 `tauri-agent/scripts/build-sidecar.mjs`。

**变更来源：** GitHub Releases v0.80.0 / v0.80.1 / v0.80.2（https://github.com/earendil-works/pi/releases）。

---

## 影响面分析

| 位置 | 包管理器 | 现状 | 处理 |
|---|---|---|---|
| `extensions/package.json` | bun（`bun.lock`） | `pi-ai` / `pi-coding-agent` `^0.79.10` | 改钉子 → `^0.80.2` |
| `cli/package.json` | npm（`package-lock.json`，shrinkwrap） | `pi-coding-agent ^0.79.6` | 改钉子 → `^0.80.2` |
| 30+ 个 `extensions/*/package.json` | — | `"*"` | 不改（跟随 hoist） |
| `tauri-agent/` | pnpm | 不直接依赖 pi-* | 不改（重建 sidecar 即可） |
| `embedding/` | — | 不依赖 pi-* | 无影响 |

**构建链关键点：** `tauri-agent/scripts/build-sidecar.mjs` 会在 `cli/` 和 `extensions/` 各跑一次 `npm install`，再 `bun build ./src/main.ts --compile` 把 pi 运行时 + 扩展全量打进单文件二进制。因为是 bun 全量打包（不走 cli `package.json` 里 esbuild 的 `--external`），所以 0.80.0 的"扩展加载器把 pi-ai root 别名到 compat"那条豁免在本项目**不生效**——必须显式改 import。

## 破坏性变更 × 实际影响（已逐条核对源码）

**必须迁移（0.80.0：pi-ai 旧全局 API 移到 `@earendil-works/pi-ai/compat`）**

| 文件 | 现状 | 受影响符号 |
|---|---|---|
| `extensions/_shared/oneshot.ts:60` | `await import("@earendil-works/pi-ai")` | `completeSimple` / `streamSimple` |
| `extensions/_shared/summarize.ts:74` | `await import("@earendil-works/pi-ai")` | `completeSimple` |
| `extensions/_shared/llm.ts:66` | `await import("@earendil-works/pi-ai")` | `completeSimple` |
| `extensions/_shared/llm.ts:8` | `import type { Context, Model } from "@earendil-works/pi-ai"` | `Context` / `Model`（随 model API 迁移，大概率需改） |
| `extensions/_shared/summarize.ts:6` | `import type { Model } from "@earendil-works/pi-ai"` | `Model` |

**仅需 typecheck 验证（大概率仍在 root，按报错再定）**：`StringEnum`（`debug-tools` / `fable-behavior` / `todo`）、`AssistantMessage` / `TextContent`（`agent-mode`）。

**确认无影响（误报，勿动）**：
- `extensions/tts/tts.ts:113` 的 `"api-key"` 是 Azure HTTP header，与 0.80.2 改的 `ApiKeyCredential.type`（`"api-key"` → `"api_key"`）无关；项目读 auth 走 `ModelRegistry.getApiKeyAndHeaders()`（pi 内部），不受影响。
- 0.80.2 重命名的 `ExecutionEnvExecOptions → ShellExecOptions`、0.80.0 移除的 `pi-ai/base` 与 `pi-agent-core/base` 入口——项目均未使用。

## 回滚

`git checkout -- extensions/package.json cli/package.json extensions/bun.lock cli/package-lock.json extensions/_shared/oneshot.ts extensions/_shared/summarize.ts extensions/_shared/llm.ts`，再在 `extensions/` `bun install`、`cli/` `npm install`，重建 sidecar。

## 风险

- `streamSimple` / `Model` / `Context` 的确切导出位置以任务 4 的 typecheck 为权威依据。
- pi-coding-agent shrinkwrap 会顺带升 `@anthropic-ai/sdk` / `openai` / `@aws-sdk/*` 等传递依赖（已打进二进制，风险低，靠任务 5/6 兜底）。
- `extensions/` 同时存在 `bun.lock` 与构建脚本里的 `npm install`，属既有现象，本次不一并整改。
- Node：`cli` 要求 `>=22.5.0`，0.80 已弃 node20（`legacy-node20` 停在 0.74.2），当前环境满足。
