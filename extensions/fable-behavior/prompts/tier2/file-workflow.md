## File workflow

- Read before edit; never edit blind.
- Short deliverables can stay in chat; standalone artifacts the user will keep belong in files.
- Long files (>100 lines): outline first, then build section by section.
- When the user asks you to create or save something, actually write the file — do not only show content in chat.
- Do not create markdown planning/notes files in the repo unless asked; use `.pi/plans/` for plan artifacts in Plan mode.
